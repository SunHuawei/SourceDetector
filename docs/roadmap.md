# SourceDetector 增长与发现优化规划（站点 + 渠道）

> 范围说明：本文件仅为规划，不直接修改 Chrome 商店文案与商店素材。

## 1) 站点（GitHub Pages）优化

目标：让首次访问者在 30 秒内理解价值、入口和可信度。

- 保持 English 为默认首页，中文/日文作为语言分支
- 首页固定展示：安装入口、核心价值、目标人群、隐私说明、FAQ、反馈入口
- 在页头和页脚保留统一导航（商店、GitHub、隐私、语言切换）
- 维护 `docs/README.md` 作为站点运维手册，降低后续协作成本

阶段建议：

- P0：结构与链接完整（已覆盖）
- P1：补「更新日志/版本动态」入口页
- P2：补「使用场景案例」页并建立内部链接

## 2) 多语言策略（EN / ZH / JA）

目标：覆盖核心受众语言，提升自然搜索与分享转化。

- English 为唯一默认入口（`/`），ZH/JA 为并列二级入口（`/zh/`, `/ja/`）
- 三语页面信息架构保持一致，避免内容漂移
- 页面头部维护 `canonical + hreflang`，减少搜索引擎语言混淆
- README 多语言文件与站点多语言入口同步维护

阶段建议：

- P0：三语页面互链 + README 互链
- P1：关键 FAQ 三语对齐
- P2：新增第 4 语言前，先建立翻译术语表与审校流程

## 3) GitHub 发现与转化优化

目标：把仓库访问转化为安装、试用和反馈。

- `README.md` 顶部固定三语入口 + 站点入口 + 隐私入口
- 在 README 显示：功能概览、适用人群、安装路径、反馈路径
- 在 `docs/` 维护公开 roadmap，持续透明化迭代方向
- 规范 Issue 模板（Bug / Feature / Question）以提升反馈质量

阶段建议：

- P0：README 链接与入口统一
- P1：补 issue templates 与 discussions 指引
- P2：按版本维护 changelog 与里程碑复盘

## 4) Chrome 商店（仅规划，不改文案）

目标：提升商店页面点击后的安装转化率和信任度。

建议动作（后续在商店后台执行）：

- 标题、副标题与首屏截图传达「source map + 泄漏检测」核心价值
- 首屏三要素：检测范围、证据视图、导出能力
- 与站点/README 保持术语一致，减少认知断层
- 明确隐私承诺（本地优先、无账号、核心流程不依赖远端）

阶段建议：

- P1：先统一叙事与素材脚本
- P2：再做 A/B 方向测试（不频繁改动）

## 5) 截图 / 视频素材规划

目标：减少“看不懂产品做什么”的流失。

- 截图最小集（建议 5 张）：
  1. 首页价值图（能力全景）
  2. Source map 发现结果
  3. 泄漏扫描结果
  4. 证据详情视图
  5. ZIP 导出与结果管理
- 演示视频（30-60 秒）：
  - 3 步流程：访问网站 -> 发现/扫描 -> 查看证据/导出
  - 结尾给出安装和反馈入口

阶段建议：

- P1：先做截图，视频后置
- P2：补短视频并按渠道复用

## 6) FAQ 体系规划

目标：减少重复问答和安装前疑虑。

优先 FAQ：

- 是否会上传数据到服务器？
- 能检测哪些泄漏类型？
- 适合哪些人使用？
- 如何反馈误报/漏报？
- 如何导出并复现证据？

阶段建议：

- P0：站点 FAQ 基础版
- P1：README FAQ 摘要 + 站点详细 FAQ
- P2：按真实 issue 高频问题迭代

## 7) SEO / GEO（搜索与生成式答案可发现性）

目标：提升自然检索曝光和 AI 回答引用概率。

- SEO 基础：
  - 每个语言页有独立 title/description
  - 维护 `canonical + hreflang`
  - 结构化数据保留 SoftwareApplication
- GEO 基础：
  - 文案采用问题-答案结构（FAQ 可直接引用）
  - 关键能力描述统一（source map 发现、泄漏检测、证据导出）
  - 公开 roadmap 与隐私承诺，增强可验证性与引用稳定性

阶段建议：

- P0：完成页面元信息与结构化数据
- P1：补充更细粒度 FAQ 与场景词
- P2：根据搜索词与 issue 反馈持续微调

## 8) 反馈入口规划

目标：把“看到问题”快速转为“可处理反馈”。

- 主入口：GitHub Issues
- 次入口：README 与站点 FAQ 指向 Issues
- 反馈建议模板字段：
  - 目标站点/环境
  - 复现步骤
  - 期望结果 vs 实际结果
  - 可公开的脱敏证据

阶段建议：

- P0：统一入口链接
- P1：模板化字段
- P2：按标签建立 triage 节奏

## 9) 后续优先级（建议执行顺序）

### P0（本轮已落地基础）

- GitHub Pages 多语言结构与互链完成
- README / docs / 页面入口统一
- 隐私与反馈入口贯通

### P1（1-2 周）

- 补齐截图最小集
- 增加 FAQ 深化与 issue templates
- 增加 changelog/更新动态入口

### P2（2-4 周）

- 发布 30-60 秒演示视频
- 增加场景案例页（研究者/开发者）
- 基于搜索词与反馈迭代 SEO/GEO 文案结构
