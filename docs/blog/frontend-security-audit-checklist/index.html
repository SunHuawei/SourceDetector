Frontend security audit checklist

A useful frontend security audit starts by reviewing what a public website ships to the browser: source maps, JavaScript bundles, public configuration, suspicious strings, and other client-side exposure clues.

Checklist

  • Check whether source maps are exposed
  • Review production JavaScript bundles
  • Scan for suspicious key and token patterns
  • Look for public config clues and internal endpoints
  • Preserve evidence before reporting or remediation

What not to assume

Not every exposed artifact is a vulnerability. The audit process should separate harmless public data from higher-risk leakage candidates.
